The Mandela Bible

A memory-led restoration of the King James Bible

Download the MVP (PDF, 1,684 pages) Markdown edition GitHub

What is this?

Generations memorized scripture — and what they remember does not match the text on the page. Wineskins, not bottles. The lion lying down with the lamb. Straight is the path. Forgive us our trespasses. The Mandela Bible treats those shared memories as witness testimony and restores the King James text accordingly, using the discipline of textual criticism.

This is the MVP edition: the complete 66-book text with 3,657 documented restorations applied — owner-reviewed memory-led repairs, the documented public memory testimony of the TSBC Scribe database (398 memories, 249 residue images of the pre-change text), and a full rare-word audit — every word appearing only 1–2 times in the KJV was reviewed against 13 witness translations and, where a period-authentic common alternative exists, restored to it. Every changed verse is marked, every original reading is preserved in the Restoration Appendix, and every decision traces to documented evidence. Nothing is changed silently.

The method

  1. Memory leads. Remembered readings — corroborated across unrelated rememberers — are the primary evidence. No written text can veto a corroborated memory.
  2. Artifacts second. The text is audited for internal seams: words with no pre-1611 attestation, emoticon-shaped punctuation, grammar alien to Early Modern English, statistical outliers.
  3. All texts advisory. Period witnesses (Wycliffe 1382, Tyndale 1526, Geneva 1599), the Greek and Hebrew sources, and Strong's lexicons inform phrasing and suspicion — they never overrule memory.
  4. Every change is accountable. Each restoration in this edition carries a confidence score and a citation trail — an owner-reviewed approval or a named external testimony source (such as the TSBC Scribe engine) — so any reading can be revisited and reversed individually.

Open project

The entire pipeline is open: the database builds from numbered, idempotent scripts; the memory evidence, review decisions, and corroboration reports are version-controlled; and the restored text is fully diffable against the base. Explore the code, the evidence files, and the decision log on GitHub.

Contributing

Anyone can create a github account and open issues. Please follow the instructions. Look at the following to determine if a verse should be updated, even more so if you disagree with a memory that already exist and has updated a verse. This is a memory led project and while memories before the mandela effect are the primary resources, other factors are taken into consideration.

  1. Consider the rarity of the word you are replacing and word you believe to be accurate. (IE. couch, which is actually crouch and attested for by other translations. Before the mandela effect, KJV had no mispelled words known. It is possible crouch isn't accurate either though since it only appears twice otherwise in the KJV, but is the better replacement until a better word is found that would have been used when the KJV was written)
  2. Consider other translations and how the verse you want to fix is being used in other translations. While these translations also have been affected by the mandela affect, clues can also be gathered as to how the original verse was written in the KJV before the mandela effect.
  3. Consider the period authenticity. Not only should the word your adding be authentic to Early Modern English during the time the KJV was written, it should also refelect the period in which Jesus was living or the time before that. Early Modern English
  4. Memory co-oberation. If more than one person agrees with your memory, it helps to attest to the change you are presenting.

Database

Having the databse is not necessary to contribute, you can contribute to white list or with memories and residuals.
